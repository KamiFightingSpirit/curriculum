Whack-a-Mole


Goal: Make a GUI program using Swing in a Model-View-Controller design architecture.

The game shows 24 buttons and the user has to click on the "mole". The game is timed so the user can get their "whack rate".


  1. You will need to create FOUR classes to complete this program. If you create all the classes before you start coding, it is easier to code and test each part as you go.

    • a Runner


      This class will contain a main method and create a View object to start the program. You will need this object to test the rest of your code.

    • a Model


      • the number of holes
      • the max number of whacks allowed
      • the number of moles
      • the score of how many were whacked
      • the time of the first whack ( use new Date() )
      • the time of the last whack ( use new Date() here too )
      • add the method below to the model
      public String whackRate() { 
    return "Your whack rate is "+
            ((timeAtEnd.getTime() - timeAtStart.getTime()) / 1000.00 / score)+
            " moles per second.";
}

    • a View


      The View object needs a member variable to hold a controller (because it will need a mouse listener to be added to the buttons).

      Place code to build a frame into a method (makeFrame). The frame will need one button for every hole in the game. NOTE: because this GUI has more than one component, you must use a JPanel. All of the buttons need a mouse listener.

      Use a for loop to add the correct number of buttons to the panel, and use a random number to pick which of the buttons will display text that says “mole!”.

      [Hint: a new GridLayout(8,3) was used to display the buttons in rows (like the image below)]



      Provide a method that disposes of the existing frame before recalling makeFrame to provide a new mole location.

    • a Controller


      This class must implement MouseListener methods so it can process mouse clicks.

      It needs member variables for the view and model.

      Make a constructor for this class. The constructor will initialize the the model variable (call the model's constructor). It should get a view object as its parameter and use that to initialize the view variable (check the view class to make sure it is passing itself to this controller).

      In the mouseClicked method, check which button was clicked (getSource). The text on the button will tell you if it is the mole. If they click on a button other than “mole!”, update the counts in the model, and use this speak() method to tell them they missed. If they click the mole button, update the counts in the model and say "ouch"! 
      void speak(String words) { 
    try { 
        Runtime.getRuntime().exec("say " + words).waitFor();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

      Check the model after each mouse click to find out when the game is over and tell them the score and their whack rate before ending the game. If the game is not over, call a method on the view to draw a new mole location.
