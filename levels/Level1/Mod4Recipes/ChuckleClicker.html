Chuckle Clicker


Goal: Make a GUI program using Swing in a Model-View-Controller design architecture.

The program will have 2 buttons. One says “Joke” the other says “Punchline”. When the user clicks on the Joke button, they will see a joke. When they click Punchline, they will see the punchline.


    If you need help with any of these steps, take a look at the MVC recipes you completed in Module 0.

    All the classes for this program should be created in the chuckle_clicker package.

    You will need to create FOUR classes to complete this program. If you create all the classes before you start coding, it is easier to code and test each part as you go.

    • a Model


      The model needs two String member variables for a joke and a punchline.
      It will also need two public methods. One that returns the joke, the other the punchline.

    • a View


      The View object needs a member variable to hold a controller because it will need a MouseListener to be added to the buttons.

      Place code to build a frame into a method (makeFrame). The View will need 2 buttons, and a label. NOTE: because this GUI has more than one component, you must put the buttons and label onto a JPanel, then add the panel to the JFrame.

      Remember to set the frame to be visible, to exit on close, and to pack itself around the components to be the right size. You must add a MouseListener (the controller) to each of the buttons.

      Make a constructor for this class. In the constructor, first initialize the controller member variable. Next, call the method you wrote to make the frame.

      Code a public method that will set text into the View's label.

    • a Controller


      This class must implement MouseListener methods so it can process mouse clicks.

      It also needs member variables for the view and model.

      Make a constructor for this class. The constructor will initialize the the model variable (call the model's constructor). It must get a view object as a parameter and use that to initialize its view variable (check the view class to make sure it is passing itself to this controller).

      In the mouseClicked method, check which button was clicked (getSource). Get the text from the model (joke or punchline) and pass it to the view to set the text on its label.

    • a Runner


      This class should contain a main method that creates a View object to start the program. You will need this object to test the rest of your code.

    • Now run the program to test it
