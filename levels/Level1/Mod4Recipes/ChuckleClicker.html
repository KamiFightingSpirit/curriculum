Chuckle Clicker


Goal: Make a GUI program using Swing in a Model-View-Controller design architecture.

The program will have 2 buttons. One says “Joke” the other says “Punchline”. When the user clicks on the Joke button, they will see a joke. When they click Punchline, they will see the punchline.


  1. You will need to create FOUR classes to complete this program. If you create all the classes before you start coding, it is easier to code and test each part as you go.

    • a Runner


      This class will contain a main method and create a View object to start the program. You will need this object to test the rest of your code.

    • a Model


      The model needs two String member variables for a joke and a punchline.
      It will also need two public methods. One that returns the joke, the other the punchline.

    • a View


      The View object needs a member variable to hold a controller (because it will need a mouse listener to be added to the buttons).

      Place code to build a frame into a method (makeFrame). The frame will need 2 buttons, and a label. NOTE: because this GUI has more than one component, you must put the buttons and label onto a JPanel, then add the panel to the JFrame.

      Remember to set the frame to be visible, to exit on close, and to pack itself around the components to be the right size. You must add a mouse listener to each of the buttons (the controller).

      Make a constructor for this class. First initialize the constructor member variable. Next, place a call to the method you wrote to make the frame.

      Code a public method that will change the label's text.

    • a Controller


      This class must implement MouseListener methods so it can process mouse clicks.

      It needs member variables for the view and model.

      Make a constructor for this class. The constructor will initialize the the model variable (call the model's constructor). It should get a view object as its parameter and use that to initialize the view variable (check the view class to make sure it is passing itself to this controller).

      In the mouseClicked method, check which button was clicked (getSource). Get the right text from the model (joke or punchline) and pass it to the view to be displayed on its label.
  2. *Optional. Customize your GUI to make it more interesting with setText, setSize, etc.
