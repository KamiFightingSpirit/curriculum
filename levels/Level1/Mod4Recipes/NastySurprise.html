Nasty Surprise


Goal: Make a program that uses Model-View-Controller (MVC) design architecture.

This program will have a window (frame) with 2 buttons. One says “Trick” the other says “Treat”. When the user clicks on the Trick button, they will see a picture of a cute puppy. When they click Treat, they will see a scary image.


    If you need help with any of these steps, take a look at the MVC recipes in Module 0

    All the classes for this program should be created in the nasty_surprise package.

    Create FOUR classes to complete this program. If you create all the classes before you start coding, it is easier to code and test each part as you go.

      Make the Model


    • Create a class for the Model.
    • Add two String variables that are initialized to the Internet addresses of two pictures (use a browser to find images, and "copy image address").
    • Create two public methods. One that returns the puppy Icon, and another that returns the scary Icon. Here is some code you can use to make an Icon (image) from an Internet address (String).
         private Icon getPictureFromTheInternet(String imageUrl) {
        Icon icon = null;
        try {
            URL url = new URL(imageUrl);
            icon = new ImageIcon(url);
        } catch (MalformedURLException e) {
            e.printStackTrace();
        }
        return icon;
    }

      • Make the View


    • Create a class for the View.
    • Add a method that will make the View. It should create a frame, a panel, two buttons ("trick" and "treat") and a label (for the picture). Remember to set the frame to be visible, to exit on close, and to pack the frame so it will be the right size.
    • Add a member variable for the controller
    • Make a constructor for this class. In the constructor, initialize a controller member variable. Next, call the method you just wrote to make the frame.
    • Make a public method that will change the picture shown on the label (setIcon). You will also need to pack the frame every time the picture changes in case it is a different size.

      • Make the Controller


    • Create a class for the Controller. This class must implement MouseListener methods so it can process mouse clicks.
    • Add member variables for a view and a model.
    • Make a constructor for this class. The constructor will initialize the the model variable (call the model's constructor). It must get a view object as a parameter and use that to initialize its view variable (check the view class to make sure it is passing itself to this controller).
    • In the mouseClicked method, check which button was clicked (getSource). Get the right picture from the model (trick or treat) based on which button was pressed, and send it to change the picture.
    • Now you have finished the Controller, go back to the View and add a MouseListener (the controller) to each of its buttons.

      • Make a Runner


    • Create a Runner class.
    • This class should contain a main method that creates a View object to start the program. You will need this object to test the rest of your code.

    • Now run the program to test it
