Nasty Surprise


Goal: Make a GUI program using Swing in a Model-View-Controller design architecture.

The program will have 2 buttons. One says “Trick” the other says “Treat”. When the user clicks on the Trick button, they will see a picture of a cute puppy. When they click Treat, they will see a scary image.


    If you need help to get started, take a look at the recipes that you already completed in module 0


  1. You will need to create FOUR classes to complete this program as follows:

    • a Model

      has String variables that contain the addresses of two pictures on the Internet (use "copy image address" to get the address of an image).

      The model will also need two methods. One that returns an Icon that contains the "trick", and another that contains the "treat". The model also needs the code below to make an Icon from an address (URL).
         private Icon getPictureFromTheInternet(String imageUrl) {
        Icon icon = null;
        try {
            URL url = new URL(imageUrl);
            icon = new ImageIcon(url);
        } catch (MalformedURLException e) {
            e.printStackTrace();
        }
        return icon;
    }

    • a View

      needs a frame, a panel, 2 buttons, and a label. NOTE: because this GUI has more than one component, you must add a JPanel to your JFrame and put the buttons and label onto the panel!

      The View must provide a method that will change the picture shown on the label and must also have a variable to hold a controller (because it will need a mouse listener on the buttons).

    • a Controller

      must be a mouse listener so it can process button presses. The controller acts as the go-between so needs to have variables for the view and model. The controller will retrieve a picture from the model based on which button was pressed, and send it to the view to be displayed on the label.

    • a Runner

      will contain the main method and create the View object to start the program
