How to Contribute to a Git Project

1. Clone the repository


2. Create a new branch


Click on the "Current Branch" tab then click the "New Branch" button.


3. Name the branch


Enter the name for the new branch. It should reflect the feature or bug fix that you plan to implement.


4. Implement the changes


Add or change the code necessary to complete the feature or bug fix.


5. Commit the changes


When the code has been updated, commit the changes like you normally would. Make sure you are still in your new branch. Also make sure to leave a descriptive commit message. Other people will be relying on your commit messages.


6. Publish the changes


Publish the commit to the repository. This will add your new branch to repo on github.com.


7. Start the Pull Request


We have to now signal that we want our changes to be merged in with the master code. Go to the "Current Branch" tab and click "Pull Request". Then click the link that reads "create a pull request".


8. Create the Pull Request


After completing step 7, you should have been presented with a github page that reads "Open a pull request". Varify all the information is correct and click "Create pull request".


9. Merge Pull Request


If there are no reviews required you can merge the pull request by clicking the green "Merge pull request" button.


However, more often than not, your pull request will need to be reviewed by another developer. If you are reviewing somebody else's pull request, you can click the link that reads "Add your review".


11. Approve Changes


Never approve changes to code without testing it first! Once you have tested the changes and are happy with them, you can check the "Approve" option and then click "Submit review".


Once all the required reviews have been completed, you can then merge the pull request by clicking "Merge pull request".


12. Merge Complete


If everything was merged successfully, you can delete the branch you created in step 2.
